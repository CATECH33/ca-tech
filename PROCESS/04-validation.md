# 04 — Validation & Recette

## Objectif
Vérifier que le livrable est conforme au brief, fonctionnel et prêt pour la mise en ligne.

## Tests internes (avant présentation client)

### Tests fonctionnels
- [ ] Navigation complète (tous les liens fonctionnent)
- [ ] Formulaires : envoi, validation, messages d'erreur
- [ ] Boutons et CTA : tous cliquables et mènent au bon endroit
- [ ] Recherche (si applicable)
- [ ] Panier et paiement (e-commerce)
- [ ] Espace client / authentification (si applicable)
- [ ] Emails transactionnels reçus correctement

### Tests responsive
- [ ] Mobile (360px, 390px, 414px)
- [ ] Tablette (768px, 1024px)
- [ ] Desktop (1280px, 1440px, 1920px)
- [ ] Vérification sur appareil réel (pas seulement émulateur)

### Tests cross-browser
- [ ] Chrome (dernière version)
- [ ] Firefox (dernière version)
- [ ] Safari (dernière version)
- [ ] Edge (dernière version)
- [ ] Safari iOS
- [ ] Chrome Android

### Tests performance
- [ ] Lighthouse Performance > 90
- [ ] Lighthouse Accessibility > 90
- [ ] Lighthouse Best Practices > 90
- [ ] Lighthouse SEO > 90
- [ ] Temps de chargement < 3s
- [ ] Images optimisées (WebP, lazy loading)
- [ ] Pas de ressources bloquantes

### Tests SEO
- [ ] Balises title uniques par page
- [ ] Meta descriptions uniques par page
- [ ] H1 unique par page
- [ ] Hiérarchie Hn respectée
- [ ] Balises alt sur toutes les images
- [ ] Schema.org implémenté
- [ ] Sitemap XML généré
- [ ] robots.txt correct
- [ ] Open Graph et Twitter Cards

### Tests sécurité
- [ ] HTTPS actif
- [ ] Formulaires protégés (honeypot, validation)
- [ ] Pas de données sensibles exposées
- [ ] Headers de sécurité configurés

### Tests contenu
- [ ] Aucune faute d'orthographe
- [ ] Textes lorem ipsum remplacés
- [ ] Images définitives en place
- [ ] Mentions légales présentes
- [ ] Politique de confidentialité présente

## Présentation au client

### Format
- Visioconférence (30-45 min)
- Démonstration en direct sur le staging
- Parcours de chaque page / fonctionnalité
- Démonstration responsive (mobile/tablette)

### Collecte des retours
- Le client liste ses retours par écrit (email ou document partagé)
- Chaque retour doit être précis (page, section, modification souhaitée)
- Délai de retour client : 5 jours ouvrés maximum

## Révisions

### Tour 1
- Intégration de tous les retours du client
- Mise à jour du staging
- Notification au client pour re-vérification
- Délai : 2-3 jours ouvrés

### Tour 2 (dernier inclus)
- Retours complémentaires du client
- Corrections finales
- Délai : 1-2 jours ouvrés

### Révisions supplémentaires
- Au-delà de 2 tours : facturation au tarif horaire
- Devis complémentaire envoyé avant toute intervention

## Validation finale
- Le client confirme par écrit que le livrable est validé
- Email de validation ou signature du PV de recette
- Facturation du solde (70%)
- Passage à l'étape 05-livraison dès réception du paiement
